# Web panel (`platform serve`)

Date: 2026-07-23
Status: approved by the owner — deliberately overrides the "no daemon/API/UI" rule

## Decision

The owner has chosen to add a full web UI that performs every platform
operation, behind password auth. This reverses the original v1/v2 stance
("the only way in is SSH"). It is a v3 feature, built now at the owner's
explicit request, with the security burden accepted.

## Security model (accepted risk)

The panel runs on the VPS and needs the Docker socket to act, which is
root-equivalent. Mitigations:

- Served only over HTTPS via Traefik (`panel.<BASE_DOMAIN>`), like the other UIs.
- A single operator password (`PANEL_PASSWORD`, generated by install.sh, stored
  in infra/.env). Login exchanges it for a signed session cookie (HMAC-SHA256
  over an expiry, keyed by a server secret). All `/api/*` routes except login
  require a valid cookie.
- No user accounts, no registration — one operator.
- The panel is off by default: it only runs if you start the `panel` compose
  service. `install.sh` sets it up but the operator can remove the service.

## Architecture

- New `platform serve` subcommand: a stdlib `net/http` server. No new Go
  dependencies (crypto/hmac, encoding/json, embed only).
- Frontend: a small single-page app embedded with `go:embed` (vanilla
  HTML/CSS/JS) — no npm, no build step. Keeps the single-binary property.
- The server reuses the existing internals (loadApp, deploy, previousSuccess,
  replicaState, provisionPostgres, …). Shared app-creation logic is extracted
  from `cmdNew` into `createApp` so the CLI and API share one path.
- Deployed as a compose service `panel` running `platform serve`, with the repo
  bind-mounted (so it reads/writes apps/) and the Docker socket mounted.

## API

- `POST /api/login {password}` → sets session cookie; `POST /api/logout`.
- `GET  /api/apps` → list with state/tag/replicas/domain.
- `GET  /api/apps/{name}` → detail + deployment history.
- `POST /api/apps {name,image,domain,port,health,provisionDB,provisionRedis}`.
- `POST /api/apps/{name}/deploy {tag}` · `/rollback` · `/scale {replicas}`.
- `GET/PUT /api/apps/{name}/env` (list / set-unset).
- `DELETE /api/apps/{name} {keepFiles}`.

Long operations (deploy/rollback) run synchronously for v1; the UI shows a
spinner and the result. Streaming logs is a later refinement.

## Testing

- Unit: session token sign/verify (valid, tampered, expired); the auth
  middleware (401 without cookie); JSON (de)serialization of the app list.
- Live: deploy/scale/rollback/env/remove an app end-to-end from the browser on
  the VPS, behind HTTPS + login.

## Out of scope (v1)

Streaming deploy logs, multi-user/roles, metrics charts (OpenObserve covers
that), editing app.yaml fields other than via the create form + scale/env.
